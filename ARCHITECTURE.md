# Архитектура BunkerAPI

## Назначение

**BunkerAPI** — веб-приложение ASP.NET Core 9 (минимальный хостинг через top-level `Program.cs`), которое отдаёт HTTP API. Основная бизнес-функция на данный момент — генерация случайной «карты» персонажа из статического JSON с наборами строк (профессия, здоровье, хобби и т.д.).

## Структура решения

| Путь | Назначение |
|------|------------|
| `BunkerAPI.sln` | Файл решения Visual Studio / `dotnet sln` |
| `BunkerAPI/` | Единственный проект — веб-API |
| `BunkerAPI/BunkerAPI.csproj` | SDK `Microsoft.NET.Sdk.Web`, целевая платформа `net9.0` |
| `BunkerAPI/Program.cs` | Точка входа: регистрация сервисов, конвейер HTTP, запуск хоста |
| `BunkerAPI/Controllers/` | Контроллеры MVC API (`[ApiController]`) |
| `BunkerAPI/Services/` | Интерфейсы и реализации прикладной логики |
| `BunkerAPI/Models/` | DTO и модели для десериализации / ответов API |
| `BunkerAPI/Data/game-data.json` | Статические списки для генерации карты |
| `BunkerAPI/Properties/launchSettings.json` | Профили запуска (URL, окружение, `launchUrl` для Swagger) |
| `BunkerAPI/appsettings*.json` | Конфигурация логирования и хоста (по умолчанию шаблонные значения) |

Сборочные артефакты (`bin/`, `obj/`) и пользовательские настройки IDE перечислены в `.gitignore`.

## Слои и зависимости

Проект следует простой схеме **контроллер → сервис → файл данных**:

1. **Контроллеры** принимают HTTP-запросы, не содержат разборов JSON с диска и не реализуют генерацию случайных значений — только вызывают зарегистрированные сервисы и формируют HTTP-ответ.
2. **Сервисы** инкапсулируют работу с данными и правилами (загрузка `game-data.json`, выбор случайных элементов).
3. **Модели** описывают контракт ответа (`Card`) и форму документа в JSON (`GameDataDocument`).

Внешние пакеты:

- **Swashbuckle.AspNetCore** — OpenAPI и UI Swagger (включён только в окружении `Development`).

## Поток запроса (пример: случайная карта)

1. Клиент вызывает `GET /api/Card`.
2. `CardController` обращается к `ICardService.GetRandomCard()`.
3. `CardService` использует заранее загруженный в память `GameDataDocument` и для каждого списка выбирает одну строку через `Random.Shared`.
4. Ответ сериализуется в JSON (имена свойств в **camelCase** — поведение по умолчанию для контроллеров ASP.NET Core).

## Загрузка и размещение данных

- Файл **`Data/game-data.json`** лежит рядом с проектом; в `BunkerAPI.csproj` для него задано **`CopyToOutputDirectory`**, чтобы при `dotnet run` и публикации копия оказалась в выходном каталоге (например `bin/Debug/net9.0/Data/`).
- `CardService` в конструкторе получает `IWebHostEnvironment` и читает файл по пути `ContentRootPath/Data/game-data.json`. Десериализация выполняется **System.Text.Json** с политикой имён **`SnakeCaseLower`**, чтобы ключи JSON (`professions`, `health_conditions`, …) соответствовали свойствам C# (`Professions`, `HealthConditions`, …).
- Сервис зарегистрирован как **`Singleton`**: файл читается один раз при первом создании экземпляра; последующие запросы не обращаются к диску.

## Конечные точки API (контроллеры)

| Маршрут | Контроллер | Описание |
|---------|------------|----------|
| `GET /hello` | `HelloController` | Простая проверка: строка «Привет мир». |
| `GET /api/Card` | `CardController` | JSON-объект `Card` с шестью случайными полями из `game-data.json`. |

В режиме разработки доступны **`/swagger`** и **`/swagger/v1/swagger.json`**.

## Конвейер HTTP (`Program.cs`)

Порядок после `Build()`:

- в **Development**: `UseSwagger`, `UseSwaggerUI`;
- `UseAuthorization` (политики авторизации по умолчанию не настроены — заготовка под будущее расширение);
- `MapControllers()` — маршрутизация к контроллерам.

Глобальных middleware для CORS, аутентификации или кастомного логирования пока нет.

## Возможные направления развития

Примеры без изменения текущей базовой схемы: вынос путей к данным в конфигурацию, кэширование с перезагрузкой, отдельные эндпоинты по типам данных, валидация и версионирование API, модульные тесты для `CardService` с подменой файла или in-memory JSON.
